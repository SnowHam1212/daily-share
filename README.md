# daily-share 設計ドキュメント

## 1. 技術スタック

| 領域 | 技術 | 備考 |
|---|---|---|
| Frontend | React + TypeScript | 型安全、チーム開発向き |
| BaaS | Supabase | PostgreSQL + Realtime + Auth が揃う |
| DB | PostgreSQL（Supabase内） | リレーショナル、スケーラブル |
| Realtime | Supabase Realtime（WebSocket） | 位置情報のリアルタイム共有 |
| マップ | Leaflet.js / MapLibre | OSS・無料 |
| ホスティング | Vercel | GitHub連携、プレビューデプロイ対応 |
| 認証 | Supabase Auth | メール／Googleログイン |

## 2. ユースケース一覧

### アクター
- **メンバー**：チームに所属する一般ユーザー
- **管理者**：チームを作成・管理するユーザー（メンバーでもある）

### UC一覧

| UC ID | ユースケース名 | アクター | 概要 |
|---|---|---|---|
| UC-01 | アカウント登録 | メンバー | メールアドレスまたはGoogleアカウントで新規登録する |
| UC-02 | ログイン／ログアウト | メンバー | 登録済みアカウントでログイン・ログアウトする |
| UC-03 | チーム作成 | 管理者 | 新しいチームを作成し、招待リンクを発行する |
| UC-04 | チーム参加 | メンバー | 招待リンクからチームに参加する |
| UC-05 | 予定を追加する | メンバー | タイトル・日時・場所を入力してカレンダーに予定を登録する |
| UC-06 | 予定を確認する | メンバー | カレンダー上でチームメンバー全員の予定を閲覧する |
| UC-07 | 予定を編集・削除する | メンバー | 自分が登録した予定を更新または削除する |
| UC-08 | 位置情報を共有する | メンバー | 現在地をリアルタイムでチームメンバーに公開する |
| UC-09 | 位置情報の共有を停止する | メンバー | 共有をオフにして現在地を非公開にする |
| UC-10 | メンバーの位置を確認する | メンバー | マップ上でオンラインメンバーの現在地を確認する |
| UC-11 | プロフィールを編集する | メンバー | 表示名・アバター画像を変更する |
| UC-12 | メンバーを管理する | 管理者 | チームメンバーの招待・除外を行う |

## 5. MVP 案

### MVPのコンセプト

> **「ログインしてすぐ使える、チームの今日を共有するアプリ」**
> 認証・チーム管理・権限まわりを最小化し、カレンダーと位置情報の共有体験を最速で届ける。

### MVP の前提・割り切り

| 項目 | MVP の方針 |
|---|---|
| 予定の公開範囲 | **全公開**（チーム内の全メンバーが全員の予定を閲覧可能） |
| 位置情報の公開範囲 | **全公開**（チーム内の全メンバーが全員の位置を閲覧可能） |
| チーム数 | 1ユーザー = 1チームのみ（複数チーム対応はv2以降） |
| 認証 | メール認証のみ（Googleログインはv2以降） |
| プロフィール | 表示名のみ（アバター画像はv2以降） |
| 予定の表示 | 週表示のみ（月表示はv2以降） |
| 管理者権限 | なし（全員が同一権限） |

### MVP に含む機能

#### 認証・ユーザー

| 機能ID | 機能名 | 対応F |
|---|---|---|
| M-01 | メール認証登録・ログイン | F-01 |
| M-02 | ログアウト | F-03 |
| M-03 | 表示名の設定 | F-04（一部） |

#### チーム

| 機能ID | 機能名 | 対応F |
|---|---|---|
| M-04 | チーム作成 | F-05 |
| M-05 | 招待リンク発行・参加 | F-06, F-07 |

#### カレンダー・予定

| 機能ID | 機能名 | 対応F |
|---|---|---|
| M-06 | 予定追加（タイトル・日時のみ） | F-09（一部） |
| M-07 | 予定一覧表示（週表示） | F-10（一部） |
| M-08 | 予定削除 | F-13 |

#### 位置情報・マップ

| 機能ID | 機能名 | 対応F |
|---|---|---|
| M-09 | 位置情報共有 ON/OFF | F-14 |
| M-10 | リアルタイム位置送信 | F-15 |
| M-11 | メンバー位置のマップ表示 | F-16 |

### MVP に含まない機能（v2以降）

| 機能 | 理由 |
|---|---|
| Googleログイン | 実装コストに対してMVP検証に不要 |
| アバター画像 | Storage設定が必要、表示名で代替可能 |
| 予定の場所・メモ入力 | コア体験（予定共有）に不要 |
| 月表示カレンダー | 週表示で最低限の体験は満たせる |
| 予定編集 | 削除→再登録で代替可能 |
| メンバー除外（管理者） | 権限設計が複雑になる |
| メンバー別フィルタ | 全公開のMVPでは不要 |
| オフラインメンバー非表示 | 共有ONの人だけ表示する設計で自然に解決 |
| 複数チーム対応 | UX複雑化を避ける |

### MVP 画面一覧（最小構成）

| 画面 | 説明 |
|---|---|
| ログイン / 新規登録画面 | メール認証、表示名入力（1画面に統合） |
| チームセットアップ画面 | 初回のみ：チーム作成 or 招待URLから参加 |
| カレンダー画面（週表示） | チーム全員の予定表示・自分の予定追加・削除 |
| マップ画面 | チーム全員の現在地表示・位置共有トグル |
